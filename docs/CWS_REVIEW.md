# Notes for Chrome Web Store reviewers

## Functionality

1. Run `npm run stage:chrome`, load `build/chrome` as an unpacked extension,
   and open a Gmail draft.
2. Enter `$\frac{1}{2}$` in the message body.
3. Click the ∑ button next to Gmail's **Bold** control.
4. Confirm that a rendered PNG is inserted into the draft.
5. Press Backspace immediately after the image and confirm that the original
   `$\frac{1}{2}$` source is restored.

## Permissions and data

- Access to `https://mail.google.com/*` lets the content script add the
  compose-toolbar control, read explicitly delimited formulas, and replace
  them with images.
- `offscreen` lets the Manifest V3 service worker create a hidden packaged
  document with the `BLOBS` reason. The document converts packaged MathJax SVG
  output into a PNG; it has no visible interface and does not broaden host
  access.
- The extension has no analytics, telemetry, advertising, or
  extension-operated service.
- MathJax code and font data are packaged with the extension. Rendering makes
  no CDN, remotely hosted code, or compilation-service request.

The service worker creates the offscreen renderer only after a Gmail content
script requests it and closes it after five idle minutes. Data handling and
Limited Use commitments are documented in the repository's
[privacy policy](../PRIVACY.md).

## Source and vendored code

Run the following from the matching source archive with Node.js 22.9 or newer:

```sh
npm ci --ignore-scripts
npm run validate
```

`package-lock.json` pins the exact official `@mathjax/src` and
`@mathjax/mathjax-newcm-font` 4.1.2 packages. The repository's vendor script
copies the reviewed runtime files byte-for-byte from those packages.
`artifacts.lock.json` records the component versions, sources, sizes, and
SHA-256 digests of vendored runtime files. Validation reproduces
and checks those files, exercises local rendering, and compares the Chrome ZIP
with its staged tree.

The vendored renderer is not hand-edited. No executable code is downloaded or
evaluated at runtime. The Chrome submission is
`dist/tex-for-gmail-chrome-<version>.zip`; it contains no WebAssembly, source
maps, tests, package manager cache, or `node_modules` tree.

The CSP permits `self` plus four exact SHA-256 hashes for deterministic style
elements generated by the pinned MathJax runtime; it does not permit
`unsafe-inline`. Browser smoke testing fails on any CSP violation.
