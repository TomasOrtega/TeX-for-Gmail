# Notes for Mozilla reviewers

## Functionality

1. Load the extension and open a Gmail draft.
2. Enter `$\frac{1}{2}$` in the message body.
3. Click the ∑ button next to Gmail's **Bold** control.
4. Confirm that a rendered PNG is inserted into the draft.
5. Press Backspace immediately after the image and confirm that the original
   `$\frac{1}{2}$` source is restored.

## Permissions and data

- `https://mail.google.com/*` lets the content script add the compose-toolbar
  control, read explicitly delimited formulas, and replace them with images.
- The extension declares no data collection. It has no analytics, telemetry,
  or extension-operated service.
- MathJax code and font data are packaged with the extension. Rendering makes
  no CDN, remote-code, or compilation-service request.

The background page is non-persistent. MathJax is loaded only after a render
request, and the browser can unload the page after the last Gmail connection
closes.

## Source and vendored code

The canonical reproduction environment is Ubuntu 24.04 x86-64, Node.js 22.9
or newer, and npm 11.17.0. The build is JavaScript-only and does not require a
native compiler, Firefox, Chrome, or network access after `npm ci`.

```sh
npm ci --ignore-scripts
npm run reproduce:firefox
```

`package-lock.json` pins the exact official `@mathjax/src` and
`@mathjax/mathjax-newcm-font` 4.1.2 packages. The repository's vendor script
copies the reviewed runtime files byte-for-byte from those packages.
`artifacts.lock.json` records the component versions, sources, sizes, and
SHA-256 digests of vendored runtime files. The reproduction command checks
those files, runs the complete coverage gate and Firefox linter, scans and
builds the Firefox package, and compares its ZIP with the staged tree. The
cross-browser `npm run validate` command additionally runs Chrome rendering
and live dependency-advisory checks in CI; those are not needed to reproduce
the Firefox artifact.

The vendored renderer is not hand-edited. Its exact npm packages contain the
readable upstream TypeScript and JavaScript source and are restored by
`npm ci`; the source archive includes the lockfile, reproducible vendor script,
authored integration source, and third-party license.

The submitted extension ZIP is
`dist/tex-for-gmail-firefox-<version>.zip`. It contains no WebAssembly, remote
scripts, source maps, tests, package manager cache, or `node_modules` tree.

The CSP permits `self` plus four exact SHA-256 hashes for deterministic style
elements generated by the pinned MathJax runtime; it does not permit
`unsafe-inline`. Browser smoke testing fails on any CSP violation.

## Reviewed upstream lint warnings

Mozilla's `addons-linter` 10.8.0 reports ten static warnings in the exact,
unmodified MathJax 4.1.2 runtime:

- `DANGEROUS_EVAL` reports the upstream `Function` constructor in
  `begingroup.js`, `boldsymbol.js`, and `tex-svg.js` once each.
- `UNSAFE_VAR_ASSIGNMENT` reports seven upstream `innerHTML` assignments in
  `tex-svg.js`.

The `Function` call is Webpack's fallback for browsers without `globalThis`;
supported Firefox versions take the `globalThis` branch, and the extension CSP
also disallows string evaluation. The reported `innerHTML` assignments belong
to MathJax menu and accessibility paths that this renderer disables. User TeX
is converted to SVG, serialized, and rasterized into a PNG before insertion.

`npm run lint` does not ignore these files or disable warnings. It consumes the
JSON lint report, requires the exact warning codes and locations, and verifies
each warned file against its SHA-256 digest in `artifacts.lock.json`. Any
warning in authored extension code, any new or missing warning, any lint error
or notice, and any change to the reviewed MathJax files fails validation.
